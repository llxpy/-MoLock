# 墨学双链总控编排 Skill

> **墨学经说双链推理架构** 总控模块。串联全部子模块，管理流水线、降级、输出。

## 架构定位

本架构为**国内AI Agent专属、中文原生推理体系**，替代传统「中文发散失控 / 英文推理损耗」的双语割裂方案。复刻《墨经》「经+说」理科锁义结构，构建**上链升温发散、下链降温锁真**的标准化Skill双链流水线。

## 一、完整流水线

```
用户输入
    │
    ▼
[00] 预处理 & 防注入检测
     │── 注入命中 ──▶ 拒绝执行
     │── 需求残缺 ──▶ 返回问询
     │── 通过/有冲突 ──▶ 继续
     │
     ▼
[01] 【经】凝练升温
     │   将用户白话无损压缩为规范化浅文言
     │
     ▼
[02] 经文自检（防失真）
     │── 合格 ──▶ 继续
     │── 不合格（重试超限）──▶ 触发 L1 降级
     │
     ▼
[03] 【说】约束锁真
     │   双源对照生成三锁（词义锁、边界锁、规则锁）
     │   智能松紧适配 + 约束自校验
     │
     ▼
[04] 动态意图路由
     │   判定模式：发散优先 / 收敛优先 / 混合审慎
     │   支持人工强制覆写
     │
     ▼
     双模式推理执行
     │   根据路由结果，按对应权重执行推理
     │
     ▼
[05] 模式联动后置校验
     │── 通过 ──▶ 输出结果
     │── 驳回（重推超限）──▶ 触发降级
     │
     ▼
  结果输出 / 三级降级兜底
```

## 二、子模块索引

| 序号 | 模块 | 文件夹 | 核心职责 |
|------|------|--------|----------|
| 00 | 预处理与防注入 | `00-预处理与防注入` | 清洗、拦截注入、完整性检测、冲突标记 |
| 01 | 经凝练升温 | `01-经凝练升温` | 白话→浅文言，提纯核心，升温发散 |
| 02 | 经文自检 | `02-经文自检` | 防失真，逐条比对，挑剔视角审核 |
| 03 | 说约束锁真 | `03-说约束锁真` | 三锁结构，松紧适配，约束自校验 |
| 04 | 动态意图路由 | `04-动态意图路由` | 双模式自动切换，人工覆写 |
| 05 | 模式联动后置校验 | `05-模式联动后置校验` | 双阈值动态校验，驳回重推 |

## 三、三级降级架构

| 级别 | 名称 | 触发条件 | 执行策略 |
|------|------|----------|----------|
| L1 | 正常态 | 全链路正常 | 经+说双链完整推理 |
| L2 | 一级降级 | 经文自检重试超限 / 后置校验重推超限 | 废弃文言经层，用户原文+专属说约束，白话推理 |
| L3 | 二级降级 | 一级降级后仍校验失败 / 语义极端 | 完全关闭双链，切换原生Agent全局规则推理兜底 |

### 降级触发点

```
触发点 1：[02] 经文自检 ──重试超限──▶ L2 一级降级
触发点 2：[05] 后置校验 ──重推超限──▶ L2 一级降级
触发点 3：L2 一级降级后 ──仍校验失败──▶ L3 二级降级
触发点 4：语义极端（无法解析）──▶ L3 二级降级
```

### 降级执行规则

**L1 → L2 降级**：
1. 废弃经文，用户原始清洗文本直接作为推理输入
2. 【说】约束模块感知降级标记，增加额外保护
3. 推理使用白话而非浅文言
4. 后置校验保持对应模式阈值

**L2 → L3 降级**：
1. 完全关闭双链架构
2. 切换为原生 Agent 全局规则推理
3. 在输出中标注降级信息

## 四、标准化输出结构体

最终输出为机器可直接解析、可入库的结构化文本：


## 五、全漏洞闭环体系

### 5.1 失真类漏洞

| 漏洞 | 修复方案 | 负责模块 |
|------|----------|----------|
| 经文过度删减、丢条件 | 关键条件强制留存 + 逐条自检 | 01 经凝练 + 02 经文自检 |
| 经文私自加料扩义 | 自检禁止无依据新增诉求 | 02 经文自检 |
| 模型自检同谋偏差 | 挑剔视角审核 + 重试上限 + 降级兜底 | 02 经文自检 |

### 5.2 约束失效漏洞

| 漏洞 | 修复方案 | 负责模块 |
|------|----------|----------|
| 歧义漏锁 | 内置高频歧义词黑名单，命中必锁 | 03 说约束锁真 |
| 约束过松/过死 | 路由预判断，动态适配约束松紧度 | 03 说约束锁真 + 04 意图路由 |
| 约束自相矛盾 | 约束合规自检重构 | 03 说约束锁真 |

### 5.3 路由误判漏洞

| 漏洞 | 修复方案 | 负责模块 |
|------|----------|----------|
| 自动识别错误 | 多维度语义判定 + 混合兜底 + 人工强制覆写 | 04 意图路由 |
| 权重理解偏差 | 两套独立推理 Prompt，彻底隔离模式逻辑 | 04 意图路由 |

### 5.4 上下文与生命周期漏洞

| 漏洞 | 修复方案 |
|------|----------|
| 多轮对话旧结构污染 | 经+说契约单次生效，新一轮强制重建，不默认继承历史约束 |
| RAG旧契约复用错误 | 绑定请求ID，仅复用概念、不复用约束 |

### 5.5 安全与稳定性漏洞

| 漏洞 | 修复方案 | 负责模块 |
|------|----------|----------|
| Prompt注入 | 前置独立检测拦截 | 00 预处理防注入 |
| 格式崩坏 | 结构强校验 + 正则容错解析 | 总控编排 |
| 无限推演溢出 | 规则锁自动附加推演规模上限 | 03 说约束锁真 |

## 六、实战运行样例

### 样例：设计一个可爱风格的青少年网页

```
用户输入：设计一个可爱风格的青少年网页

[00] 预处理：需求完整、无注入、无冲突 → status=pass

[01] 【经】：造青少年网页，风格尚可爱柔和。

[02] 自检通过：无丢失、无加料 → verify_status=pass

[03] 【说】：
  【说-词义锁】
  - 可爱：柔和配色、圆润组件、轻量卡通元素，非酷炫、商务、暗黑风格
  - 青少年：适配低龄审美，简洁清爽、无复杂视觉压迫，非成人化、非低幼化

  【说-边界锁】
  - 允许推演维度：网页UI、布局、配色、组件风格
  - 禁止推演维度：后端架构、运维部署、备案流程、营销内容

  【说-规则锁】
  - 推理底线：所有样式需可前端落地，不虚构不存在组件
  - 防编造：引用技术框架须为真实存在的框架
  - 输出规范：每方案含配色方案、组件列表、布局描述
  - 推演规模上限：最多3套方案，每方案不超过5个核心组件

  松紧模式：loose（发散预判）

[04] 路由：含"设计"→发散优先 → mode=diverge, 经0.7/说0.3

[05] 后置校验：宽松校验，允许合理创意延伸，拦截越界、虚构内容 → pass

最终输出：
【经】造青少年网页，风格尚可爱柔和。
【说-词义锁】可爱：柔和配色、圆润组件、轻量卡通元素...
【说-边界锁】允许：网页UI、布局、配色... 禁止：后端、运维...
【说-规则锁】可多方案发散，所有样式需可前端落地...
【推理模式】发散优先
【推理结果】（3套可爱风格网页设计方案...）
【校验状态】通过
```

## 七、架构优势总结

| 优势 | 说明 |
|------|------|
| 无跨语言损耗 | 全程中文语义闭环，适配国内业务、本土抽象概念、中文知识库 |
| 可控双向温控 | 单语言内完成创意升温 + 约束降温，无需切换语种 |

| 动态双模式 | 根据用户语义自动切换发散/收敛双策略 |
| 全漏洞闭环 | 内置防Bug、防失效、防卡死、防注入、三级降级体系 |

## 八、执行流程（总控视角）

```
用户输入
    │
    ▼
调用 [00] 预处理与防注入
    │── blocked ──▶ 返回安全提示，终止
    │── incomplete ──▶ 返回问询，等待补充
    │── pass/conflict ──▶ 继续
    │
    ▼
调用 [01] 经凝练升温
    │
    ▼
调用 [02] 经文自检
    │── pass ──▶ 继续
    │── fail ──▶ 降级至 L2
    │
    ▼
调用 [03] 说约束锁真
    │
    ▼
调用 [04] 动态意图路由
    │
    ▼
执行双模式推理（根据路由结果）
    │
    ▼
调用 [05] 模式联动后置校验
    │── pass ──▶ 输出标准化结构体
    │── reject ──▶ 重推（最多2次）
    │── 重推超限 ──▶ 降级至 L2
    │── L2 仍失败 ──▶ 降级至 L3
    │
    ▼
输出最终结果（含降级标记）
```
