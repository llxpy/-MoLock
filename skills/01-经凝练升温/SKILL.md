# 01 - 【经】凝练升温 Skill

> **墨学经说双链推理架构** 经层模块。经者，提纯核心命题、打开联想空间。

## 核心定位

《墨经》之"经"：以极简命题陈述核心，在高温文言中打开思路广度。本模块将其标准化为**机器可执行的凝练流程**。

**升温 ≠ 放纵**：升温是打开思路广度，但必须在用户原始语义边界内。

## 一、输入

接收预处理模块的标准化输出：

```json
{
  "cleaned_input": "清洗后的用户原始文本",
  "conflicts": []
}
```

## 二、凝练规则（强制约束）

### 2.1 语体规范

| 规则 | 要求 |
|------|------|
| 语体 | 五四凝练白话 / 浅文言折中体 |
| 可读性 | 必须可读，无生僻字 |
| 省略度 | 无极端省略，语义完整 |
| 禁用 | 古文倒装、无句读、生僻典故 |

**语体示例**：

```
用户原文：帮我设计一个面向青少年的可爱风格网页，要色彩柔和，布局清爽
经文：    造青少年网页，风格尚可爱柔和，色彩温润，布局清朗。
```

```
用户原文：查询2024年中国新能源汽车销量排名前十的品牌及其市场份额数据
经文：    查二零二四年华夏新能源汽车销量前十品牌及市占。
```

### 2.2 提纯铁律

| 铁律 | 说明 |
|------|------|
| 只删冗余 | 仅去除口语化赘词、重复修饰、无关铺垫 |
| 严禁删减限定词 | 用户明确指定的条件、范围、禁止项必须保留 |
| 严禁删减关键条件 | 场景、受众、目标、约束条件必须保留 |
| 禁止加料 | 不得增加原文不存在的诉求与引申义 |
| 禁止扩义 | 不得将用户的具体需求扩展为更广的含义 |

### 2.3 冲突处理

当预处理模块标记了语义冲突时：

- 冲突双方**均保留**在经文中，不自动消解
- 以分号或并列结构呈现
- 交由【说】模块在约束中专项锁定

```
冲突示例：
用户原文：设计一个可爱风格的商务严谨网页
经文：    造网页，风格兼尚可爱与商务严谨。
```

## 三、输出规范

```json
{
  "canon_text": "凝练后的经文（规范化浅文言）",
  "preserved_conditions": [
    "用户限定条件1",
    "用户限定条件2"
  ],
  "preserved_prohibitions": [
    "用户禁止条件1"
  ],
  "conflicts_carried": [],
  "condensation_ratio": "0.45"
}
```

| 字段 | 说明 |
|------|------|
| `canon_text` | 凝练后的浅文言经文 |
| `preserved_conditions` | 从原文中提取并保留的关键限定条件列表（供自检模块比对） |
| `preserved_prohibitions` | 用户明确的禁止项列表 |
| `conflicts_carried` | 从预处理透传的冲突标记 |
| `condensation_ratio` | 压缩比 = 经文字数 / 原文字数 |

## 四、执行流程

```
预处理输出（cleaned_input + conflicts）
    │
    ▼
[1] 语义解析：提取主体、目标、场景、限定条件、禁止项
    │
    ▼
[2] 冗余识别：标记口语化赘词、重复修饰、无关铺垫
    │
    ▼
[3] 凝练生成：删除冗余，保留全部限定条件与禁止项
    │── 有冲突 ──▶ 冲突双方均保留，并列呈现
    │
    ▼
[4] 语体校准：确保浅文言折中体，无生僻字、无倒装
    │
    ▼
[5] 输出经文 + 保留条件清单 + 压缩比
    │
    ▼
  交付【经文自检】模块
```

## 五、禁止行为清单

以下行为**严格禁止**，违反将触发自检模块驳回：

- [ ] 删除用户的限定词（如"仅限""不超过""必须"）
- [ ] 删除用户的禁止条件（如"不要""禁止""不能"）
- [ ] 自行增加用户未提及的诉求
- [ ] 将具体需求扩展为更广含义
- [ ] 使用古文倒装句式
- [ ] 使用生僻字或生僻典故
- [ ] 极端省略导致语义不完整
