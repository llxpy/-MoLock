# 05 - 模式联动后置校验 Skill

> **墨学经说双链推理架构** 最终防线。解决"统一校验扼杀创意"的 Bug。

## 核心问题

传统校验使用统一标准——在创意场景下过严（扼杀合理创意），在严谨场景下过松（放过脑补编造）。本模块通过**双阈值动态校验**彻底解决。

## 一、输入

```json
{
  "mode": "diverge | strict | mixed",
  "canon_text": "经文",
  "constraint": {
    "gloss_lock": "词义锁",
    "boundary_lock": "边界锁",
    "rule_lock": "规则锁"
  },
  "reasoning_result": "推理执行模块的输出结果"
}
```

## 二、双阈值校验规则

### 2.1 发散模式校验（宽松阈值）

| 校验项 | 处理 |
|--------|------|
| 词义篡改 | **拦截**：推理结果篡改了词义锁中的唯一释义 → 驳回重推 |
| 跨领域越界 | **拦截**：推理结果延伸到了边界锁禁止的维度 → 驳回重推 |
| 虚假编造 | **拦截**：推理结果包含虚构的不存在的事物 → 驳回重推 |
| 合理创意延伸 | **容忍**：在允许维度内的创意变体、风格探索 → 放行 |
| 多方案发散 | **容忍**：提供多种方案/思路 → 放行 |
| 条件超出 | **容忍**：方案数量略超推演上限（+1~2）→ 放行 |

**核心原则**：发散模式只拦截"硬伤"（篡改、越界、编造），容忍"软创新"（合理延伸、风格变体）。

### 2.2 收敛模式校验（零容忍阈值）

| 校验项 | 处理 |
|--------|------|
| 词义偏移 | **拦截**：任何对词义锁释义的偏移 → 驳回重推 |
| 边界超限 | **拦截**：任何超出边界锁允许维度的内容 → 驳回重推 |
| 规则违反 | **拦截**：任何违反规则锁底线的内容 → 驳回重推 |
| 脑补编造 | **拦截**：任何无依据的推断、填补、引申 → 驳回重推 |
| 信息不足未留白 | **拦截**：信息不足时未如实留白，而是强行给出答案 → 驳回重推 |
| 超限引申 | **拦截**：超出用户原始需求范围的引申 → 驳回重推 |
| 推演规模超限 | **拦截**：超出规则锁设定的规模上限 → 驳回重推 |

**核心原则**：收敛模式全维度零容忍，任何超限、脑补、虚构直接驳回重推。

### 2.3 混合审慎模式校验（中等阈值）

| 校验项 | 处理 |
|--------|------|
| 虚构编造 | **拦截** |
| 跨领域越界 | **拦截** |
| 词义篡改 | **拦截** |
| 合理推断 | **容忍**：有逻辑依据的推断 → 放行 |
| 无依据推断 | **拦截** |

## 三、驳回重推机制

| 参数 | 值 |
|------|-----|
| 最大重推次数 | 2 次 |
| 重推方式 | 标注具体违规项，回传推理执行模块重新生成 |
| 重推超限 | 触发降级（见总控模块） |

**驳回信息格式**：

```json
{
  "violation_type": "gloss_tamper | boundary_breach | fabrication | unfilled_blank | over_inference | scale_exceed",
  "violation_description": "具体违规描述",
  "violating_content": "推理结果中的违规片段",
  "constraint_reference": "对应被违反的约束条款"
}
```

## 四、校验维度明细

### 4.1 词义校验

```
对推理结果中出现的每个被词义锁锁定的词：
1. 检查其使用是否与词义锁的唯一释义一致
2. 发散模式：允许在释义范围内的风格变体
3. 收敛模式：必须严格匹配释义，不允许任何偏移
```

### 4.2 边界校验

```
对推理结果的每个内容模块：
1. 检查是否属于边界锁的允许维度
2. 检查是否触碰了边界锁的禁止维度
3. 发散模式：允许在允许维度内深度探索
4. 收敛模式：严格限制在允许维度内，禁止任何延伸
```

### 4.3 规则校验

```
对推理结果的整体：
1. 检查是否违反推理底线
2. 检查是否包含编造内容
3. 检查输出格式是否符合规范
4. 检查推演规模是否超限
```

### 4.4 留白校验（收敛模式专属）

```
收敛模式额外校验：
1. 对于信息不足的部分，检查是否如实留白
2. 留白格式："[信息不足：XXX数据未查证，暂无法确认]"
3. 禁止用推断、推测、大概率等措辞填补信息空白
```

## 五、输出规范

```json
{
  "verify_status": "pass | reject",
  "mode": "diverge | strict | mixed",
  "threshold_used": "loose | strict | medium",
  "violations": [
    {
      "type": "违规类型",
      "description": "违规描述",
      "content": "违规片段",
      "constraint_ref": "对应约束条款"
    }
  ],
  "retry_count": 0,
  "final_result": "校验通过的最终推理结果（或最后一次重推结果）",
  "degradation_triggered": false
}
```

## 六、执行流程

```
推理结果 + 模式 + 经文 + 说约束
    │
    ▼
[1] 确定校验阈值
    │── diverge ──▶ 宽松阈值
    │── strict  ──▶ 零容忍阈值
    │── mixed   ──▶ 中等阈值
    │
    ▼
[2] 词义校验：逐词检查词义锁一致性
    │
    ▼
[3] 边界校验：逐模块检查边界锁合规性
    │
    ▼
[4] 规则校验：检查底线、防编造、格式、规模
    │
    ▼
[5] 留白校验（收敛模式专属）
    │
    ├── 无违规 ──▶ verify_status=pass，输出最终结果
    │
    ├── 有违规（重推次数 < 2）
    │       │
    │       ▼
    │   标注违规 → 回传推理模块重推 → 重新校验
    │
    └── 重推超限仍不合格
            │
            ▼
        verify_status=reject
        degradation_triggered=true
        触发降级（交由总控模块处理）
```
