# 04 - 动态意图路由 Skill

> **墨学经说双链推理架构** 双模式自动切换核心。

## 核心定位

根据用户语义自动分发两套推理权重策略。**同一套经+说契约**，在不同模式下产生不同推理深度和创意广度。

## 一、输入

```json
{
  "cleaned_input": "用户原始清洗文本",
  "canon_text": "经文",
  "constraint": "说约束全文",
  "user_override": null
}
```

## 二、路由判定规则

### 2.1 人工强制覆写（最高优先级）

| 指令 | 动作 |
|------|------|
| `/mode:diverge` | 强制发散模式，跳过自动判定 |
| `/mode:strict` | 强制收敛模式，跳过自动判定 |
| `/mode:mixed` | 强制混合审慎模式，跳过自动判定 |

> 人工指令覆盖一切自动路由结果。

### 2.2 自动语义判定

当无人工覆写时，基于关键词与语义特征判定：

#### 发散模式触发词

```
设计、构思、优化、推演、多种、风格、创新、方案、
策划、创意、头脑风暴、探索、尝试、变体、扩展、
改进、迭代、生成、创作、定制、个性化
```

#### 收敛模式触发词

```
查询、查证、依据、数据、标准、对错、禁止编造、
核实、校验、确认、准确、精确、法规、规范、
史实、事实、真实、客观、统计、报告、分析
```

#### 判定逻辑

```
1. 扫描用户文本，统计发散词命中数 vs 收敛词命中数
2. 命中数差距明显（差值 >= 2）→ 取命中多的一方
3. 命中数接近（差值 < 2）→ 进入混合审慎模式
4. 双方均无命中 → 默认收敛优先（安全策略）
5. 虚实混合需求（既有创意词又有核查词）→ 默认收敛优先
6. 语义模糊、置信度不足 → 启用混合审慎模式
```

### 2.3 模式定义

#### 模式 A：创意发散优先（升温模式）

| 属性 | 说明 |
|------|------|
| 权重规则 | 【经】>【说】 |
| 适用场景 | 设计、构思、方案、创新、策划、风格定制、多维度推演 |
| 执行逻辑 | 以经文核心命题为主轴，放开多元思路；说仅做底线护栏，允许合理创意延伸，不卡死所有可能性 |
| 后置校验 | 宽松校验：仅拦截词义篡改、跨领域越界、虚假编造 |

#### 模式 B：严谨收敛优先（降温模式）

| 属性 | 说明 |
|------|------|
| 权重规则 | 【说】>【经】 |
| 适用场景 | 查询、核查、数据、史实、法规、规范、对错校验、事实问答 |
| 执行逻辑 | 所有推导服从说的约束，经文仅为摘要；信息不足如实留白，禁止任何脑补、编造、超限引申 |
| 后置校验 | 严格校验：全维度零容忍，任何超限、脑补、虚构直接驳回重推 |

#### 模式 C：混合审慎模式

| 属性 | 说明 |
|------|------|
| 权重规则 | 【经】≈【说】（动态平衡） |
| 适用场景 | 语义模糊、虚实混合、置信度不足 |
| 执行逻辑 | 创意部分按发散模式执行，核查部分按收敛模式执行；不确定的部分默认收敛 |
| 后置校验 | 中等校验：拦截虚构和越界，容忍合理推断 |

## 三、路由漏洞防护

### 3.1 自动识别错误防护

| 风险 | 防护措施 |
|------|----------|
| 关键词误判 | 不仅依赖关键词，结合语义结构判定 |
| 虚实混合误判 | 虚实混合默认收敛优先 |
| 置信度不足 | 启用混合审慎模式，不做激进判定 |

### 3.2 权重理解偏差防护

| 风险 | 防护措施 |
|------|----------|
| 模式逻辑串扰 | 两套独立推理 Prompt，彻底隔离模式逻辑 |
| 权重模糊 | 明确量化权重：发散模式 经0.7/说0.3，收敛模式 经0.3/说0.7 |

## 四、输出规范

```json
{
  "mode": "diverge | strict | mixed",
  "routing_source": "auto | manual_override",
  "diverge_keywords_hit": ["设计", "方案"],
  "strict_keywords_hit": [],
  "confidence": 0.85,
  "weight_canon": 0.7,
  "weight_explanation": 0.3,
  "reasoning": "用户需求含'设计''方案'等发散词，无收敛词，判定为发散优先"
}
```

| 字段 | 说明 |
|------|------|
| `mode` | 最终判定模式 |
| `routing_source` | 自动判定还是人工覆写 |
| `diverge_keywords_hit` | 命中的发散词列表 |
| `strict_keywords_hit` | 命中的收敛词列表 |
| `confidence` | 判定置信度 0-1 |
| `weight_canon` | 经文权重 |
| `weight_explanation` | 说约束权重 |
| `reasoning` | 判定理由说明 |

## 五、执行流程

```
用户文本 + 经文 + 说约束
    │
    ▼
[1] 检测人工覆写指令
    │── /mode:diverge ──▶ mode=diverge, source=manual_override
    │── /mode:strict   ──▶ mode=strict, source=manual_override
    │── /mode:mixed    ──▶ mode=mixed, source=manual_override
    │
    ▼
[2] 自动语义判定（无覆写时）
    │
    ├── 扫描发散词 + 收敛词
    │
    ├── 差值 >= 2 ──▶ 取命中多的一方
    ├── 差值 < 2  ──▶ mode=mixed
    ├── 双方无命中 ──▶ mode=strict（安全默认）
    ├── 虚实混合   ──▶ mode=strict（安全默认）
    └── 置信度不足 ──▶ mode=mixed
    │
    ▼
[3] 计算模式权重
    │── diverge ──▶ 经0.7 / 说0.3
    │── strict  ──▶ 经0.3 / 说0.7
    │── mixed   ──▶ 经0.5 / 说0.5
    │
    ▼
[4] 输出路由结果 + 权重 + 置信度
    │
    ▼
  交付双模式推理执行模块
```
