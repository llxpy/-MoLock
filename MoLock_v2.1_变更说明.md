## 二、技能包 v2.1 变更（3 个文件）

### 1. `03-说约束锁真`（explanation-constrain）→ v2.1
- **新增「否定结构强制锁定表」**：覆盖否定词库（不/没/无/非/未…）、嵌套否定识别规则（双重否定回正、三重及以上自内向外逐层还原）、范围副词（都/全/仅/只…）、量词指代（其/此/前者/后者…）。**命中必锁，自动生成逐层还原链**——直接修复"否定靠人锁"的 #1 缺口。
- **新增「§3.1 题型自适应触发」**：按 NLI / 词汇歧义 / 否定范围 / 创意 四类题型自动决定放松哪一把锁（NLI 放松边界锁、否定范围全维度严锁）。把"自适应"从设计变为部分实现。
- 接收路由模块下传的 `task_type_hint` 触发对应松紧。

### 2. `04-动态意图路由`（intent-router）→ v2.1
- **新增「嵌套否定深度识别」**：标注 `negation_depth`，向说约束模块传递 `negation_detected + negation_depth`，由其自动锁否定（不再依赖人）。
- **新增 `task_type_hint` 输出字段**：通用 / NLI / 词汇歧义 / 否定范围 / 创意，供说约束自动触发题型自适应松紧。

### 3. `总控编排`（orchestrator）→ v2.1
- **新增「[−1] 难度预路由」**：简单/低歧义题（短句、无嵌套否定、无角色陷阱）直答，仅过防注入，跳过 01–05，**省 token**（回应"显性=贵"）。
- **新增「[02]/[05] 硬卡点」**：经文自检与后置校验为不可跳过硬卡点，自动化部署时 LLM 不得省略（回应"依赖执行人守协议"）。
- JSON 输出 `version` 升为 `2.1.0`；§5.2 闭环表补充"否定结构强制锁定表"。

## 四、剩余待办（诚实标注，非本次范围）

1. **触发分类器的稳定性**：`task_type_hint` 目前靠关键词启发式，长尾题型仍可能误判。
2. **难度预路由的判定阈值**：简单/复杂的边界需实测校准，避免漏放难题进直答。
3. **自动化硬卡点的工程封装**：论文/技能描述的是"应做"，真正不可跳过需在调用层（LangGraph / Agent 框架）做结构化强制，而非仅靠 Prompt 要求。
