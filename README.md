# 墨锁（MoLock）

> 墨家「经说双链」—— 一套中文大模型语义约束推理架构，把几千年前的墨学工程语言搬进 AI。

## 一句话

**墨锁是一套 Skills 链，让 AI 用中文推理时不容易跑偏、不会脑补不存在的信息。**

---

## 问题

中文大模型有个通病：语义空间太大，越"深度思考"越容易发散。

比如 "这产品还行，只是价格不太友好"——模型可能会读出 "差评，作者不会购买"，但实际上原文根本没提到购买意愿。

英文 API 幻觉少？对。但国内开发者英文水平参差不齐，不能全换英文。

**有没有一种方法，让中文推理也能像英文一样精确？**

---

## 灵感来源

我在墨家经典《墨经》里找到了答案。

墨子那一派是先秦诸子百家里唯一的"工程师团队"——他们写光学、力学、几何，用的全是文言文。文言文表达范围极宽，但他们需要精确描述"光沿直线传播"——差一个字就全错了。

他们的解法：「经以概括，说以约束。」

- **经**：用极简的浅文言锁定核心语义
- **说**：对每个可能产生歧义的词，逐条注释约束

两千多年前的工程师就已经解决了"中文语义边界模糊"的问题。我把这套方法搬到了大模型推理上。

## 架构

```
用户输入
  │
  ▼
┌─────────────────┐
│ 00 预处理与防注入  │  规则层：清洗输入、拦截注入
└────────┬────────┘
         ▼
┌─────────────────┐
│ 01 经凝练升温    │  将白话无损压缩为浅文言，锁定核心语义
└────────┬────────┘
         ▼
┌─────────────────┐
│ 02 经文自检      │  拿原文与经文逐条比对，防止凝练失真
└────────┬────────┘
         ▼
┌─────────────────┐
│ 03 说约束锁真    │  三锁结构：词义锁 + 边界锁 + 规则锁
└────────┬────────┘
         ▼
┌─────────────────┐
│ 04 意图路由      │  自动判定发散/收敛/混合推理模式
└────────┬────────┘
         ▼
┌─────────────────┐
│ 05 正式推理      │  在约束框架内执行，拒绝脑补
└────────┬────────┘
         ▼
┌─────────────────┐
│ 06 后置校验      │  双阈值校验：发散宽松 / 收敛零容忍
└────────┬────────┘
         ▼
      输出结果
```

7 个模块，流水线串联。支持三级降级：正常 → 弃经文用白话 → 关闭双链兜底。

## 实验数据（初步）

使用 DeepSeek-v4-flash，3 道中文歧义测试题，裸调 vs 墨锁对比：

| 题号 | 陷阱类型 | 裸调幻觉 | 墨锁处理 |
|------|----------|----------|----------|
| 1 | 词汇歧义「还行」 | 滑到「差评」，脑补购买决策 | 锁定「及格」，未跨界 |
| 2 | 空占位符「外部环境变化」 | 编造"责任转移"叙事 | 明确标注「无法确定」 |
| 3 | 角色缺失「小王/老张」 | 编造完整职场权力关系剧本 | 只做语用分析，零编造 |

**3 题合计：裸调 4 处幻觉，墨锁 0 处。**

> 注：当前为小样本定性实验，更大规模的定量验证正在进行中。

## 使用方法

墨锁是基于 WorkBuddy Skills 规范构建的，7 个模块可独立使用，也可通过总控编排一键串联。

### 安装

1. 下载 `skills/` 目录下的 7 个文件夹
2. 导入到 WorkBuddy 的 Skills 面板
3. 使用时加载「墨学双链总控编排」即可自动串联全部模块

### 单独使用某个模块

每个模块都可独立加载，适合特定场景：
- 只想做 Prompt 清洗 → 只加载「00-预处理与防注入」
- 只想做歧义约束 → 只加载「03-说约束锁真」
